This page provides a list of the FHIR artifacts defined as part of this implementation guide.

Structures: Resource Profiles

These define constraints on FHIR resources for systems conforming to this implementation guide

HIV MedicationStatement - ART

A summary of the ART medication being taken by an HIV patient
HIV Patient

A patient that may be diagnosed with HIV, may have a viral load lab, and may be virally suppressed.
Observation - HIV Status

HIV status reported after applying the HIV testing algorithm. No single HIV test can provide an HIV-positive diagnosis.
Observation - Key Population

The type of key population that the client is included in
Observation - Viral Load Test

Reason for HIV viral load test

Structures: Extension Definitions

These define constraints on FHIR data types for systems conforming to this implementation guide

ART Start Type

Whether the client is ART naive or is restarting ART
Estimated Age

Estimated Age of Patient if date of birth is unavailable
Reason for HIV viral load test

Whether the viral load is being tested for routine monitoring on a set schedule or for targeted monitoring for suspected treatment failure

Terminology: Value Sets

These define sets of codes used by systems conforming to this implementation guide

ART Start Type values

Codes describing whether the client is ART naive or is restarting ART
Gender values

Codes describing the gender of the client
HIV Status values

Codes describing whether the client received a positive HIV test result
Key Population values

Codes describing the type of key population that the client is included in
Reason for HIV viral load test values

Codes describing the reason for HIV viral load test

Terminology: Code Systems

These define new code systems used by systems conforming to this implementation guide

HIV Extended Codes

Codes defined only in this Implementation Guide and require mapping to a standard semantic terminology.
